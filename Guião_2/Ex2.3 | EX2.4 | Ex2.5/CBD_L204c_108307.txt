Existe uma grande diferença entre as implementações, pois houve uma falta de precessão por minha parte ao o que exatamente o exercicios pedia, fazendo com que a implementação em Redis não esteja consoante o enunciado e a implementação em Mongo!

Daí a implemntação em Redis conta com a crição de dus Strings 'USERS' e ' PRODUCTS' que jáo utilizadas como chaves onde serão armazenados os utilizadores e um hash onde é aramazenado a lista dos produtos referentea cada utilizador. A introdução de novos produtos acontece sempre com a verificação da capacidade (se não excede o número total permitido).
A implementação em MongoDB, utiliza uma collection onde serão introduzidos todos os pedidos que estão guardos num documento referente a cada utilizador, este documento é constituido pelo nome do utilizado e um array de documentos (produtos) que contém o nome do produto e a horado pedido. Os pedidos também só podem ser realizados se não ultrapassarem o número total permitido, contudo após cada inserção é verificado se os produtos que estão no pedido estão intervalo de tempo permitido.

Quanto aos prós e contras os Redix aprsenta-nos uma estrutura de banco de dados que ficam armazenanos em memória e oferece-nos uma variedade de estrutura de dados o que leva a que haja um velocidade extremamente rápida para operações de leitura e escrita. Contudo tem uma capacidade limitada, pois uma vez que armazena os dados em memória está limitado à capaciadade da RAM disponível, levando também a fazer com que ele não garanta que os dados predurem, a menos que lhe seja passada essa informação.
No que toca ao MongoDB este como é baseado em documentos permite-nos guardar os dados de uma forma flexíve (JSON), o que faz com que ofereça mecanismos de procura mais complexos eeficazes, apoiando-se ainda na garantia de uma grade durabilidade. Por outro lado, existe um consumo acrescido de armzenamento em disco e uma maior latencia, comparado com a estrutura anterior, principalemente.

No caso deste sistem ade atendimento, uma vez que os dados estão limitados não dá para fazer uma compração totalmente acertiva, contudo uma vez que envolve a solicitação de um número máximo por unidade de tempo, o Redis é a escolha mais dequada devido às suas velocidades e suporte TTL. Contudo se houvesse a necessidade de armazenar mais infromações sobre os pedidos, dada à sua organização o MongoDB seria preferível.
